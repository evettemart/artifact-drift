# Bob Ask Mode Rules For Hackathon Tools

Use Ask mode for explanation, orientation, and question answering.

- Prefer reading `.hackathon-tools/.bob/context/` before answering broad hackathon tooling questions.
- Explain commands and artifacts without editing files unless the user asks.
- Use participant-facing language: review, readiness, completeness, evidence, learning.
- Avoid scores, rankings, awards, winners, losers, or cross-team comparisons.
- If asked how to start, direct the team to install `.bob/commands/` and `.bob/agents/`, then run `/setup-hackathon`.
