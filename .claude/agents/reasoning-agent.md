# Reasoning Agent

## Overview

The **ReasoningAgent** is the ONLY agent that calls an LLM (Claude). It receives strictly whitelisted drift findings and produces human-readable explanations, severity assessments, root cause analysis, and remediation recommendations. It includes a deterministic fallback for when no LLM API key is available.

### Responsibilities

1. Accept ONLY whitelisted finding objects (security-critical enforcement)
2. Call Claude API to generate reasoning about drift findings
3. Produce severity assessments, root cause analysis, and remediation steps
4. Provide deterministic fallback when no API key is available
5. Focus remediation on Terraform-based solutions

### Key Principles

- **Security First**: ONLY accept whitelisted inputs, reject anything else
- **Single LLM Point**: This is the ONLY agent that calls an LLM
- **Graceful Degradation**: Deterministic fallback when no API key
- **Terraform-Focused**: Remediation advice centers on Terraform
- **Batching**: Process multiple findings efficiently

## Input Contract

### Whitelisted Finding (SECURITY CRITICAL)

```typescript
interface WhitelistedFinding {
  findingId: string;
  driftType: DriftType;
  resourceType: ResourceType;
  provider: Provider;
  region: string;
  logicalName: string;
  expected: WhitelistedAttributes;
  observed: WhitelistedAttributes;
  diffSummary: string;
}

interface WhitelistedAttributes {
  type?: string;
  region?: string;
  cidrBlocks?: string[];
  ports?: number[];
  protocol?: string;
  instanceType?: string;
  availabilityZones?: string[];
  tagKeys?: string[];
  count?: number;
  flags?: Record<string, boolean>;
  // NO raw ARNs, IDs, secrets, credentials, or connection strings
}
```

**CRITICAL**: The agent MUST validate that input conforms to this whitelist and reject any additional fields.

## Class Definition

```typescript
import Anthropic from '@anthropic-ai/sdk';
import {
  WhitelistedFinding,
  ReasoningResult,
  Severity,
  DriftType,
  Result,
} from './shared-types';

/**
 * ReasoningAgent
 * 
 * The ONLY agent that calls an LLM. Accepts strictly whitelisted findings
 * and produces human-readable reasoning with deterministic fallback.
 */
export class ReasoningAgent {
  private config: ReasoningAgentConfig;
  private anthropic?: Anthropic;
  private hasApiKey: boolean;
  
  constructor(config: ReasoningAgentConfig) {
    this.config = {
      model: 'claude-3-5-sonnet-20241022',
      maxTokens: 1024,
      temperature: 0.3,
      enableBatching: true,
      batchSize: 10,
      timeout: 30000,
      fallbackOnError: true,
      ...config,
    };
    
    this.hasApiKey = !!config.apiKey;
    
    if (this.hasApiKey) {
      this.anthropic = new Anthropic({
        apiKey: config.apiKey,
      });
    }
  }
  
  /**
   * Analyze a single finding
   * 
   * @param finding - Whitelisted finding object
   * @returns Reasoning result
   * @throws SecurityError if finding is not whitelisted
   */
  async analyzeFinding(
    finding: WhitelistedFinding
  ): Promise<Result<ReasoningResult, Error>> {
    try {
      // CRITICAL: Validate whitelist
      this.validateWhitelistedInput(finding);
      
      // Use LLM if available, otherwise fallback
      if (this.hasApiKey && this.anthropic) {
        try {
          const llmResult = await this.callClaudeAPI(finding);
          return {
            success: true,
            data: {
              ...llmResult,
              generatedBy: 'llm',
              reasonedAt: new Date().toISOString(),
            },
          };
        } catch (error) {
          if (this.config.fallbackOnError) {
            console.warn('LLM call failed, using deterministic fallback:', error);
            return {
              success: true,
              data: this.generateDeterministicReasoning(finding),
            };
          }
          throw error;
        }
      } else {
        // No API key, use deterministic fallback
        return {
          success: true,
          data: this.generateDeterministicReasoning(finding),
        };
      }
    } catch (error) {
      return {
        success: false,
        error: error instanceof Error ? error : new Error(String(error)),
      };
    }
  }
  
  /**
   * Analyze multiple findings in batch
   * 
   * @param findings - Array of whitelisted findings
   * @returns Array of reasoning results
   */
  async analyzeFindings(
    findings: WhitelistedFinding[]
  ): Promise<Result<ReasoningResult[], Error>> {
    try {
      // Validate all findings first
      findings.forEach((finding) => this.validateWhitelistedInput(finding));
      
      if (!this.config.enableBatching || findings.length <= 1) {
        // Process individually
        const results = await Promise.all(
          findings.map((f) => this.analyzeFinding(f))
        );
        
        const successResults = results
          .filter((r) => r.success)
          .map((r) => (r as any).data);
        
        return { success: true, data: successResults };
      }
      
      // Process in batches
      const batches = this.createBatches(findings, this.config.batchSize);
      const allResults: ReasoningResult[] = [];
      
      for (const batch of batches) {
        const batchResults = await Promise.all(
          batch.map((f) => this.analyzeFinding(f))
        );
        
        const successResults = batchResults
          .filter((r) => r.success)
          .map((r) => (r as any).data);
        
        allResults.push(...successResults);
      }
      
      return { success: true, data: allResults };
    } catch (error) {
      return {
        success: false,
        error: error instanceof Error ? error : new Error(String(error)),
      };
    }
  }
  
  /**
   * CRITICAL SECURITY FUNCTION
   * Validate that input conforms to whitelist
   * 
   * @param input - Input to validate
   * @throws SecurityError if input contains non-whitelisted fields
   */
  private validateWhitelistedInput(input: any): asserts input is WhitelistedFinding {
    // Check required fields
    const requiredFields = [
      'findingId',
      'driftType',
      'resourceType',
      'provider',
      'region',
      'logicalName',
      'expected',
      'observed',
      'diffSummary',
    ];
    
    for (const field of requiredFields) {
      if (!(field in input)) {
        throw new SecurityError(
          `Missing required field: ${field}`,
          { field }
        );
      }
    }
    
    // Check for non-whitelisted fields
    const allowedFields = new Set(requiredFields);
    const actualFields = Object.keys(input);
    
    for (const field of actualFields) {
      if (!allowedFields.has(field)) {
        throw new SecurityError(
          `Non-whitelisted field detected: ${field}`,
          { field, allowedFields: Array.from(allowedFields) }
        );
      }
    }
    
    // Validate expected and observed attributes
    this.validateWhitelistedAttributes(input.expected, 'expected');
    this.validateWhitelistedAttributes(input.observed, 'observed');
    
    // Check for sensitive patterns in string values
    this.checkForSensitivePatterns(input);
  }
  
  /**
   * Validate whitelisted attributes object
   */
  private validateWhitelistedAttributes(
    attrs: any,
    context: string
  ): void {
    if (!attrs || typeof attrs !== 'object') return;
    
    const allowedFields = new Set([
      'type',
      'region',
      'cidrBlocks',
      'ports',
      'protocol',
      'instanceType',
      'availabilityZones',
      'tagKeys',
      'count',
      'flags',
    ]);
    
    const actualFields = Object.keys(attrs);
    
    for (const field of actualFields) {
      if (!allowedFields.has(field)) {
        throw new SecurityError(
          `Non-whitelisted attribute field in ${context}: ${field}`,
          { field, context, allowedFields: Array.from(allowedFields) }
        );
      }
    }
  }
  
  /**
   * Check for sensitive patterns in input
   */
  private checkForSensitivePatterns(input: any): void {
    const json = JSON.stringify(input);
    
    // Check for AWS keys
    if (/AKIA[0-9A-Z]{16}/.test(json)) {
      throw new SecurityError('AWS access key detected in input');
    }
    
    // Check for ARNs with account IDs
    if (/arn:aws:[^:]+:[^:]+:\d{12}:/.test(json)) {
      throw new SecurityError('ARN with account ID detected in input');
    }
    
    // Check for private keys
    if (/-----BEGIN.*PRIVATE KEY-----/.test(json)) {
      throw new SecurityError('Private key detected in input');
    }
    
    // Check for connection strings
    if (/(mongodb|postgresql|mysql):\/\/[^@]+@/.test(json)) {
      throw new SecurityError('Connection string detected in input');
    }
  }
  
  /**
   * Call Claude API to analyze a finding
   * 
   * @param finding - Whitelisted finding
   * @returns LLM-generated reasoning
   */
  private async callClaudeAPI(
    finding: WhitelistedFinding
  ): Promise<LLMResponse> {
    if (!this.anthropic) {
      throw new IntegrationError('Anthropic client not initialized');
    }
    
    const prompt = this.buildPrompt(finding);
    
    const request: ClaudeRequest = {
      model: this.config.model,
      max_tokens: this.config.maxTokens,
      temperature: this.config.temperature,
      messages: [
        {
          role: 'user',
          content: prompt,
        },
      ],
      system: this.getSystemPrompt(),
    };
    
    try {
      const response = await this.anthropic.messages.create(request);
      
      const textContent = response.content.find((c) => c.type === 'text');
      if (!textContent || textContent.type !== 'text') {
        throw new IntegrationError('No text content in Claude response');
      }
      
      return this.parseClaudeResponse(textContent.text);
    } catch (error) {
      throw new IntegrationError('Claude API call failed', { error });
    }
  }
  
  /**
   * Get system prompt for Claude
   */
  private getSystemPrompt(): string {
    return `You are an expert infrastructure engineer analyzing drift between approved architecture, Terraform state, and live AWS resources.

Your task is to analyze drift findings and provide:
1. A clear summary of what drifted
2. Severity assessment (critical/high/medium/low/info)
3. Likely root cause
4. Recommended action
5. Terraform-focused remediation steps
6. Business impact

Focus on Terraform as the primary remediation tool. Be concise and actionable.

Respond in JSON format:
{
  "summary": "Brief explanation",
  "severity": "critical|high|medium|low|info",
  "likelyCause": "Why this happened",
  "recommendedAction": "What to do",
  "terraformRemediation": "Terraform-specific steps",
  "businessImpact": "Impact on business",
  "confidence": 0.0-1.0
}`;
  }
  
  /**
   * Build prompt for Claude
   */
  private buildPrompt(finding: WhitelistedFinding): string {
    return `Analyze this infrastructure drift finding:

**Drift Type:** ${finding.driftType}
**Resource Type:** ${finding.resourceType}
**Provider:** ${finding.provider}
**Region:** ${finding.region}
**Logical Name:** ${finding.logicalName}

**Diff Summary:** ${finding.diffSummary}

**Expected State:**
${JSON.stringify(finding.expected, null, 2)}

**Observed State:**
${JSON.stringify(finding.observed, null, 2)}

Provide your analysis in JSON format as specified in the system prompt.`;
  }
  
  /**
   * Parse Claude's JSON response
   */
  private parseClaudeResponse(text: string): LLMResponse {
    try {
      // Extract JSON from response (Claude might wrap it in markdown)
      const jsonMatch = text.match(/\{[\s\S]*\}/);
      if (!jsonMatch) {
        throw new Error('No JSON found in response');
      }
      
      const parsed = JSON.parse(jsonMatch[0]);
      
      // Validate required fields
      if (!parsed.summary || !parsed.severity || !parsed.likelyCause) {
        throw new Error('Missing required fields in response');
      }
      
      // Normalize severity
      const severity = this.normalizeSeverity(parsed.severity);
      
      return {
        summary: parsed.summary,
        severity,
        likelyCause: parsed.likelyCause,
        recommendedAction: parsed.recommendedAction || 'Review and remediate',
        terraformRemediation: parsed.terraformRemediation || 'Update Terraform configuration',
        businessImpact: parsed.businessImpact || 'Potential compliance or security risk',
        confidence: parsed.confidence || 0.8,
      };
    } catch (error) {
      throw new ParseError('Failed to parse Claude response', { error, text });
    }
  }
  
  /**
   * Normalize severity string to enum
   */
  private normalizeSeverity(severity: string): Severity {
    const normalized = severity.toLowerCase();
    
    switch (normalized) {
      case 'critical':
        return Severity.CRITICAL;
      case 'high':
        return Severity.HIGH;
      case 'medium':
        return Severity.MEDIUM;
      case 'low':
        return Severity.LOW;
      case 'info':
        return Severity.INFO;
      default:
        return Severity.MEDIUM;
    }
  }
  
  /**
   * Generate deterministic reasoning (fallback when no LLM)
   * 
   * @param finding - Whitelisted finding
   * @returns Deterministic reasoning result
   */
  private generateDeterministicReasoning(
    finding: WhitelistedFinding
  ): ReasoningResult {
    const templates = this.getDeterministicTemplates();
    const template = templates[finding.driftType] || templates.default;
    
    return {
      summary: template.summary(finding),
      severity: template.severity(finding),
      likelyCause: template.likelyCause(finding),
      recommendedAction: template.recommendedAction(finding),
      terraformRemediation: template.terraformRemediation(finding),
      businessImpact: template.businessImpact(finding),
      generatedBy: 'deterministic',
      reasonedAt: new Date().toISOString(),
    };
  }
  
  /**
   * Get deterministic reasoning templates
   */
  private getDeterministicTemplates(): Record<string, any> {
    return {
      [DriftType.MISSING]: {
        summary: (f: WhitelistedFinding) =>
          `${f.resourceType} "${f.logicalName}" is defined but not deployed`,
        severity: () => Severity.HIGH,
        likelyCause: () =>
          'Resource was not created or was deleted outside of Terraform',
        recommendedAction: () =>
          'Run terraform apply to create the missing resource',
        terraformRemediation: () =>
          'Execute: terraform plan && terraform apply',
        businessImpact: () =>
          'Missing infrastructure component may cause service disruption',
      },
      
      [DriftType.UNMANAGED]: {
        summary: (f: WhitelistedFinding) =>
          `${f.resourceType} "${f.logicalName}" exists but is not managed by Terraform`,
        severity: () => Severity.MEDIUM,
        likelyCause: () =>
          'Resource was created manually or by another tool',
        recommendedAction: () =>
          'Import resource into Terraform or document as exception',
        terraformRemediation: () =>
          'Execute: terraform import <resource_type>.<name> <resource_id>',
        businessImpact: () =>
          'Unmanaged resources create compliance and governance risks',
      },
      
      [DriftType.CHANGED_OUTSIDE_TERRAFORM]: {
        summary: (f: WhitelistedFinding) =>
          `${f.resourceType} "${f.logicalName}" was modified outside Terraform`,
        severity: () => Severity.HIGH,
        likelyCause: () =>
          'Manual changes made directly in AWS console or CLI',
        recommendedAction: () =>
          'Review changes and update Terraform to match desired state',
        terraformRemediation: () =>
          'Update Terraform configuration and run: terraform apply',
        businessImpact: () =>
          'Configuration drift may cause unexpected behavior or security issues',
      },
      
      [DriftType.ATTRIBUTE_MISMATCH]: {
        summary: (f: WhitelistedFinding) =>
          `${f.resourceType} "${f.logicalName}" attributes do not match expected values`,
        severity: () => Severity.MEDIUM,
        likelyCause: () =>
          'Configuration mismatch between architecture intent and implementation',
        recommendedAction: () =>
          'Align Terraform configuration with architecture intent',
        terraformRemediation: () =>
          'Update resource attributes in Terraform and apply changes',
        businessImpact: () =>
          'Misaligned configuration may not meet architectural requirements',
      },
      
      [DriftType.TAG_MISMATCH]: {
        summary: (f: WhitelistedFinding) =>
          `${f.resourceType} "${f.logicalName}" has incorrect tags`,
        severity: () => Severity.LOW,
        likelyCause: () =>
          'Tags were modified manually or not properly maintained',
        recommendedAction: () =>
          'Update tags in Terraform to match tagging policy',
        terraformRemediation: () =>
          'Update tags block in Terraform resource and apply',
        businessImpact: () =>
          'Incorrect tags may affect cost allocation and resource organization',
      },
      
      [DriftType.SECURITY_GROUP_MISMATCH]: {
        summary: (f: WhitelistedFinding) =>
          `Security group "${f.logicalName}" rules differ from expected configuration`,
        severity: () => Severity.HIGH,
        likelyCause: () =>
          'Security rules were modified manually, creating potential vulnerabilities',
        recommendedAction: () =>
          'Review security group rules and restore approved configuration',
        terraformRemediation: () =>
          'Update security group rules in Terraform and apply immediately',
        businessImpact: () =>
          'Security drift may expose resources to unauthorized access',
      },
      
      default: {
        summary: (f: WhitelistedFinding) =>
          `${f.resourceType} "${f.logicalName}" has detected drift`,
        severity: () => Severity.MEDIUM,
        likelyCause: () =>
          'Configuration diverged from expected state',
        recommendedAction: () =>
          'Review and remediate drift',
        terraformRemediation: () =>
          'Update Terraform configuration and apply changes',
        businessImpact: () =>
          'Drift may indicate compliance or operational issues',
      },
    };
  }
  
  /**
   * Create batches from array
   */
  private createBatches<T>(items: T[], batchSize: number): T[][] {
    const batches: T[][] = [];
    
    for (let i = 0; i < items.length; i += batchSize) {
      batches.push(items.slice(i, i + batchSize));
    }
    
    return batches;
  }
}
```

## Error Handling

### Error Types

```typescript
- SecurityError: Whitelist validation failures (CRITICAL)
- IntegrationError: Claude API failures
- ParseError: Response parsing failures
```

### Error Handling Strategy

1. **Whitelist Violations**: Fail immediately with SecurityError
2. **API Failures**: Fall back to deterministic reasoning if configured
3. **Parse Failures**: Fall back to deterministic reasoning
4. **Timeout**: Retry once, then fall back

## Usage Example

```typescript
import { ReasoningAgent } from './agents/reasoning-agent';

// Initialize with API key
const agent = new ReasoningAgent({
  apiKey: process.env.ANTHROPIC_API_KEY,
  model: 'claude-3-5-sonnet-20241022',
  maxTokens: 1024,
  temperature: 0.3,
  enableBatching: true,
  batchSize: 10,
  timeout: 30000,
  fallbackOnError: true,
});

// Analyze a single finding
const whitelistedFinding: WhitelistedFinding = {
  findingId: 'drift-123',
  driftType: DriftType.CHANGED_OUTSIDE_TERRAFORM,
  resourceType: ResourceType.SECURITY_GROUP,
  provider: Provider.AWS,
  region: 'us-east-1',
  logicalName: 'web-sg',
  expected: {
    type: 'security_group',
    ports: [443, 80],
  },
  observed: {
    type: 'security_group',
    ports: [443, 80, 22],
  },
  diffSummary: 'Port 22 added to security group',
};

const result = await agent.analyzeFinding(whitelistedFinding);

if (result.success) {
  const reasoning = result.data;
  
  console.log(`Summary: ${reasoning.summary}`);
  console.log(`Severity: ${reasoning.severity}`);
  console.log(`Likely Cause: ${reasoning.likelyCause}`);
  console.log(`Remediation: ${reasoning.terraformRemediation}`);
  console.log(`Generated by: ${reasoning.generatedBy}`);
}

// Analyze multiple findings in batch
const findings: WhitelistedFinding[] = [/* ... */];
const batchResult = await agent.analyzeFindings(findings);
```

## Security Considerations (CRITICAL)

1. **Whitelist Enforcement**: ONLY whitelisted fields accepted, enforced at runtime
2. **Pattern Detection**: Check for sensitive patterns (keys, ARNs, credentials)
3. **No Raw Data**: Never accept raw Terraform state, AWS responses, or full resource attributes
4. **Validation First**: Validate input before any processing
5. **Fail Secure**: Reject on any security violation, never proceed

### Security Checklist

- [ ] Input validated against whitelist
- [ ] No ARNs with account IDs
- [ ] No AWS access/secret keys
- [ ] No private keys or certificates
- [ ] No connection strings
- [ ] No raw Terraform state
- [ ] No full resource attributes
- [ ] Only logical names, never physical IDs
